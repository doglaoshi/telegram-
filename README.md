# Telegram 管理机器人

这是一个简单的 Telegram 机器人，当群成员在群聊中发送"1"时，会自动给予他们管理员权限。

## 功能特点

- 🤖 自动检测群聊中的"1"消息
- 👑 自动提升发送者为管理员
- 📝 记录所有提升操作的日志
- 🔒 安全检查：确保机器人有足够权限
- 📊 避免重复操作：检查用户当前状态

## 安装和配置

### 1. 创建 Telegram 机器人

1. 在 Telegram 中找到 [@BotFather](https://t.me/botfather)
2. 发送 `/newbot` 命令
3. 按提示设置机器人名称和用户名
4. 获取机器人的 token

### 2. 安装依赖

```bash
pip install -r requirements.txt
```

### 3. 配置机器人

1. 复制配置文件模板：
   ```bash
   cp config.json.example config.json
   ```

2. 编辑 `config.json` 文件，填入你的机器人 token：
   ```json
   {
     "bot_token": "你的机器人TOKEN"
   }
   ```

### 4. 设置机器人权限

1. 将机器人添加到你的群聊中
2. 给机器人以下管理员权限：
   - ✅ 删除消息
   - ✅ 封禁用户
   - ✅ 邀请用户
   - ✅ 固定消息
   - ✅ 添加管理员 (重要!)
   - ✅ 管理群聊
   - ✅ 管理语音聊天

## 使用方法

### 启动机器人

```bash
python bot.py
```

### 使用机器人

1. 在群聊中发送消息"1"
2. 机器人会自动检测并给你管理员权限
3. 成功后会收到确认消息

## 文件说明

- `bot.py` - 主要的机器人代码
- `config.json` - 配置文件（需要手动创建）
- `config.json.example` - 配置文件模板
- `requirements.txt` - Python 依赖包
- `bot.log` - 机器人运行日志
- `promotions.json` - 管理员提升记录

## 安全特性

- 只在群聊中工作，私聊无效
- 检查机器人权限，避免操作失败
- 避免重复提升已经是管理员的用户
- 新管理员不会获得"提升其他人"的权限，避免权限滥用
- 详细的日志记录所有操作

## 注意事项

⚠️ **重要提醒**：
- 确保机器人有"添加管理员"的权限
- 只有发送准确的"1"（不含其他字符）才会触发
- 机器人需要先成为群管理员才能提升其他用户
- 建议在测试群中先试用

## 故障排除

### 常见问题

1. **机器人没有反应**
   - 检查机器人是否在线
   - 确认 token 配置正确
   - 查看 `bot.log` 日志文件

2. **提示权限不足**
   - 确保机器人是群管理员
   - 检查机器人是否有"添加管理员"权限

3. **配置文件错误**
   - 检查 `config.json` 格式是否正确
   - 确认 token 没有多余空格

## 日志查看

机器人会自动记录运行日志到 `bot.log` 文件，包括：
- 用户操作记录
- 错误信息
- 权限检查结果

查看日志：
```bash
tail -f bot.log
```
